ワークショップの目的

GitHub Copilot は、AI によるコード補完や自動化を活用することで、開発スピードの向上や生産性の最大化を実現するツールです。しかし、「GitHub Copilot がどのように役立つのか具体的に知りたい」、「PoC（試験導入）を進める前に、実際の利用感を確かめたい」、「チームや組織でどのように Copilot を展開すればよいか分からない」といった課題を抱えている企業も少なくありません。このワークショップでは、実際に手を動かしながら GitHub Copilot の活用方法を学ぶことで、開発現場での具体的な活用イメージを持っていただける内容となっています。

内容

GitHub Copilot の基礎と活用シナリオ

GitHub Copilot の仕組みや、どのように AI がコード生成を支援するのかを解説。実際の業務シナリオに沿った活用例を紹介します。

実践ハンズオン – GitHub Copilot を使ってみる

参加者が自身の環境で GitHub Copilot を実際に操作し、コード補完、エージェントモード、コードレビュー、テスト作成、ドキュメント生成 などの機能を試します。「どこまでAIに頼れるのか？」「どんなケースで使うと効果的なのか？」といった実践的な理解を深めます。

企業での導入・活用のベストプラクティス

GitHub Copilot を企業で導入する際のポイントや、PoC の進め方や、開発チーム内でのベストプラクティスについても紹介します。
